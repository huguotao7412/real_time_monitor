校准期 10 秒 UX 流程

pipeline.py 已有 _calibration_done 逻辑（前 200 帧/10 秒）。UI 需要感知并展示这个过程。

状态流转

点击"开始" → ● 校准中 (0-10s) → ● 监测中 (校准完成)

校准中显示：环形进度条 + 提示文字

校准完成：环形消失 → 淡入波形和 BPM → 伴随短暂绿色脉冲

CalibrationOverlay 设计方案

A

环形进度条 + 文字

半透明深色遮罩上，居中显示一个圆环进度（0→100% 顺时针填充）。环内显示倒计时秒数。下方文字："正在校准，请保持静止..."。

直观、有仪式感、用户明确知道需要等多久

B

波纹扩散动效

从中心向外扩散的淡蓝色波纹（类似雷达扫描），每 2 秒扩散一圈。文字："正在搜索生命体征信号..."。

契合毫米波雷达的主题，科技感强，但不显示具体进度

DSP 异常 → 人类指令映射表

DSP 条件 当前显示 优化后显示 UI 视觉
phase_range < 0.005 信号无效 (phase_range=0.003) 未检测到微动，请确认在雷达覆盖范围内 (0.5m-1.5m) 状态栏黄色
breath_ratio < 0.03 信号无效 (breath_ratio=0.02) 信号较弱，请调整坐姿，正对雷达 SQI 红色
apnea_state = True (无特别提示，BPM 衰减) 监测到呼吸微弱或屏息状态 背景微红 + BPM 半透明
大幅体动(高频能量峰值 > 3σ) (无专门提示) 检测到体动干扰，请保持放松 状态栏橙色闪烁
